图片是网页内容的重要组成部分。正确使用图片标签不仅能让页面更美观，还能提升加载速度和���障碍体验。

## img 标签基础

`<img>` 是嵌入图片的标签，它是一个**空元素**，没有结束标签。

```html
<img src="photo.jpg" alt="风景照片" />
```

### src 属性

`src`（source）指定图片的路径：

```html
<!-- 相对路径 -->
<img src="images/photo.jpg" alt="照片" />
<img src="../assets/logo.png" alt="Logo" />

<!-- 绝对路径 -->
<img src="/images/banner.jpg" alt="Banner" />

<!-- 完整 URL -->
<img src="https://example.com/image.png" alt="外部图片" />

<!-- Data URL（内嵌图片） -->
<img src="data:image/png;base64,iVBORw0KGgo..." alt="内嵌图片" />
```

### alt 属性

`alt`（alternative）提供图片的文字替代，当图片无法显示时会显示这段文字。

```html
<img src="chart.png" alt="2024年销售数据图表，显示Q4增长30%" />
```

#### alt 的重要性

1. **无障碍访问**：屏幕阅读器会朗读 alt 文本，让视障用户了解图片内容
2. **图片加载失败**：显示替代文本而不是一片空白或破碎图标
3. **SEO**：搜索引擎通过 alt 理解图片内容

#### alt 的写法原则

| 图片类型     | alt 写法           |
| ------------ | ------------------ |
| 信息性图片   | 描述图片传达的信息 |
| 装饰性图片   | 留空 `alt=""`      |
| 链接内的图片 | 描述链接目标       |
| 图表/数据    | 描述数据含义       |

```html
<!-- 信息性图片：描述内容 -->
<img src="team.jpg" alt="团队成员在会议室讨论项目" />

<!-- ���饰性图片：alt 为空 -->
<img src="decorative-line.png" alt="" />

<!-- 链接内的图片：描述链接目的 -->
<a href="/home">
  <img src="logo.png" alt="返回首页" />
</a>

<!-- 图表：描述数据 -->
<img src="sales-chart.png" alt="销售图表：1月100万，2月150万，3月200万" />
```

**常见错误：**

```html
<!-- 错误：alt 描述文件名 -->
<img src="photo.jpg" alt="photo.jpg" />

<!-- 错误：alt 重复周围文字 -->
<p>我们的团队</p>
<img src="team.jpg" alt="我们的团队照片" />

<!-- 错误：装饰性图片有详细 alt -->
<img src="divider.png" alt="一条漂亮的分隔线" />
```

### width 和 height 属性

```html
<img src="photo.jpg" alt="照片" width="800" height="600" />
```

指定图片的宽高有两个好处：

1. **避免布局偏移**：浏览器在图片加载前就知道预留多少空间
2. **性能优化**：减少重排（reflow）

现代写法推荐同时设置 HTML 属性和 CSS：

```html
<img
  src="photo.jpg"
  alt="照片"
  width="800"
  height="600"
  class="responsive-img"
/>
```

```css
.responsive-img {
  width: 100%;
  height: auto;
}
```

HTML 的 width/height 确定宽高比，CSS 控制实际显示大小。

### title 属性

```html
<img src="info.png" alt="信息图标" title="点击获取更多信息" />
```

`title` 在鼠标悬停时显示提示文字。但它不是 alt 的替代品：

- 屏幕阅读器可能忽略 title
- 触屏设备无法触发悬停
- 键盘用户无法看到

## 响应式图片

在不同设备上显示不同尺寸的图片，可以提升加载速度和显示效果。

### srcset 属性

`srcset` 提供多个图片源，让浏览器根据设备选择：

```html
<!-- 基于像素密度 -->
<img
  src="photo.jpg"
  srcset="photo.jpg 1x, photo@2x.jpg 2x, photo@3x.jpg 3x"
  alt="照片"
/>

<!-- 基于图片宽度 -->
<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  alt="照片"
/>
```

### sizes 属性

配合 `srcset` 使用，告诉浏览器图片的显示尺寸：

```html
<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 400px"
  alt="照片"
/>
```

`sizes` 的语法是媒体查询和对应的显示宽度：

- 屏幕宽度 ≤ 600px 时，图片占 100% 视口宽度
- 屏幕宽度 ≤ 1200px 时，图片占 50% 视口宽度
- 其他情况，图片显示为 400px 宽

浏览器根据这些信息，从 `srcset` 中选择最合适的图片。

### picture 元素

`<picture>` 提供更精细的控制，可以为不同条件指定完全不同的图片：

```html
<picture>
  <!-- 深色模式 -->
  <source srcset="logo-dark.png" media="(prefers-color-scheme: dark)" />

  <!-- 小屏幕 -->
  <source srcset="photo-mobile.jpg" media="(max-width: 768px)" />

  <!-- 支持 WebP 格式 -->
  <source srcset="photo.webp" type="image/webp" />

  <!-- 默认图片 -->
  <img src="photo.jpg" alt="照片" />
</picture>
```

`<picture>` 的使用场景：

- 艺术方向（不同屏幕显示不同构图的图片）
- 格式兼容（提供 WebP/AVIF 等现代格式）
- 深色/浅色模式切换

## 懒加载

懒加载让图片在进入视口时才开始加载，可以显著提升首屏加载速度。

### 原生懒加载

```html
<img src="photo.jpg" alt="照片" loading="lazy" />
```

`loading` 属性的值：

- `lazy`：懒加载
- `eager`：立即加载（默认）

首屏图片不要使用懒加载，否则会延迟显示：

```html
<!-- 首屏图片：立即加载 -->
<img src="hero.jpg" alt="首页大图" loading="eager" />

<!-- 下方图片：懒加载 -->
<img src="product1.jpg" alt="产品1" loading="lazy" />
<img src="product2.jpg" alt="产品2" loading="lazy" />
```

### decoding 属性

```html
<img src="photo.jpg" alt="照片" decoding="async" />
```

`decoding` 控制图片解码方式：

- `async`：异步解码，不阻塞页面渲染
- `sync`：同步解码
- `auto`：浏览器决定（默认）

对于非关键图片，推荐使用 `decoding="async"`。

### fetchpriority 属性

```html
<!-- 高优先级加载 -->
<img src="hero.jpg" alt="首屏大图" fetchpriority="high" />

<!-- 低优先级加载 -->
<img src="thumbnail.jpg" alt="缩略图" fetchpriority="low" />
```

这个属性告诉浏览器资源的加载优先级，有助于优化 LCP（Largest Contentful Paint）。

## 图片格式选择

不同图片格式各有优劣：

| 格式 | 特点                             | 适用场景                   |
| ---- | -------------------------------- | -------------------------- |
| JPEG | 有损压缩，文件小                 | 照片、复杂图像             |
| PNG  | 无损压缩，支持透明               | 图标、简单图形、需要透明   |
| GIF  | 支持动画，256色限制              | 简单动画                   |
| WebP | 比 JPEG/PNG 更小，支持透明和动画 | 通用格式（需要兼容方案）   |
| AVIF | 比 WebP 更小，最新格式           | 追求极致压缩（兼容性有限） |
| SVG  | 矢量图，无限缩放                 | 图标、Logo、简单图形       |

### 使用 picture 提供格式兼容

```html
<picture>
  <source srcset="image.avif" type="image/avif" />
  <source srcset="image.webp" type="image/webp" />
  <img src="image.jpg" alt="图片" />
</picture>
```

浏览器会按顺序检查 source，使用第一个支持的格式。

## figure 和 figcaption

`<figure>` 用于包装独立的内容单元，通常是图片配上说明：

```html
<figure>
  <img src="chart.png" alt="2024年用户增长图表" />
  <figcaption>图1：2024年用户增长趋势</figcaption>
</figure>
```

`<figcaption>` 是 `<figure>` 的标题/说明，可以放在图片前面或后面：

```html
<figure>
  <figcaption>图2：团队合影</figcaption>
  <img src="team.jpg" alt="团队成员在办公室的合影" />
</figure>
```

### figure 的其他用途

`<figure>` 不只是用于图片，还可以用于：

```html
<!-- 代码块 -->
<figure>
  <pre><code>console.log('Hello')</code></pre>
  <figcaption>示例：输出 Hello</figcaption>
</figure>

<!-- 引用 -->
<figure>
  <blockquote>学而不思则罔，思而不学则殆。</blockquote>
  <figcaption>——《论语》</figcaption>
</figure>

<!-- 视频 -->
<figure>
  <video src="demo.mp4" controls></video>
  <figcaption>产品演示视频</figcaption>
</figure>
```

## 图片映射 map

`<map>` 和 `<area>` 可以在图片上定义可点击区域：

```html
<img src="world-map.png" alt="世界地图" usemap="#worldmap" />

<map name="worldmap">
  <area shape="rect" coords="0,0,100,100" href="/asia" alt="亚洲" />
  <area shape="circle" coords="200,150,50" href="/europe" alt="欧洲" />
  <area
    shape="poly"
    coords="300,100,350,200,250,200"
    href="/africa"
    alt="非洲"
  />
</map>
```

`shape` 定义区域形状：

- `rect`：矩形，coords 是"左,上,右,下"
- `circle`：圆形，coords 是"圆心x,圆心y,半径"
- `poly`：多边形，coords 是各顶点坐标

这个功能现在用得较少，因为可以用 CSS/SVG 实现更好的交互效果。

## 性能优化最佳实践

### 1. 使用正确的尺寸

不要用 CSS 缩小大图片：

```html
<!-- 不好：加载 2000px 宽的图片，显示为 200px -->
<img src="large-2000.jpg" alt="图片" style="width: 200px" />

<!-- 好：直接使用合适尺寸的图片 -->
<img src="thumbnail-200.jpg" alt="图���" style="width: 200px" />
```

### 2. 压缩图片

使用工具压缩图片：

- TinyPNG / TinyJPG
- Squoosh（Google 出品）
- ImageOptim（Mac）

### 3. 使用 CDN

图片托管在 CDN 上，可以加快加载速度：

```html
<img src="https://cdn.example.com/images/photo.jpg" alt="照片" />
```

### 4. 指定宽高

始终指定 width 和 height，避免布局偏移：

```html
<img src="photo.jpg" alt="照片" width="800" height="600" />
```

### 5. 首屏图片预加载

```html
<head>
  <link rel="preload" as="image" href="hero.jpg" />
</head>
```

### 6. 综合示例

```html
<picture>
  <source
    srcset="hero-mobile.avif 400w, hero-desktop.avif 1200w"
    type="image/avif"
    sizes="(max-width: 768px) 100vw, 50vw"
  />
  <source
    srcset="hero-mobile.webp 400w, hero-desktop.webp 1200w"
    type="image/webp"
    sizes="(max-width: 768px) 100vw, 50vw"
  />
  <img
    src="hero-desktop.jpg"
    srcset="hero-mobile.jpg 400w, hero-desktop.jpg 1200w"
    sizes="(max-width: 768px) 100vw, 50vw"
    alt="产品展示图"
    width="1200"
    height="800"
    loading="eager"
    fetchpriority="high"
    decoding="async"
  />
</picture>
```

## 无障碍最佳实践

### 有意义的 alt

```html
<!-- 信息图片：描述内容 -->
<img src="error.png" alt="错误：用户名已存在" />

<!-- 功能图片：描述功能 -->
<button>
  <img src="search.svg" alt="搜索" />
</button>
```

### 装饰图片

对于纯装饰性图片，使用空 alt 或 CSS 背景：

```html
<!-- 空 alt -->
<img src="decorative.png" alt="" />

<!-- CSS 背景更好 -->
<div class="decorated"></div>
```

### 复杂图片

对于图表、信息图等复杂图片，提供详细描述：

```html
<figure>
  <img src="chart.png" alt="用户增长趋势图" />
  <figcaption>
    图表显示2024年用户增长趋势：1月10万，6月50万，12月100万，
    增长主要来自移动端用户。
  </figcaption>
</figure>

<!-- 或使用 aria-describedby -->
<img src="chart.png" alt="用户增长趋势图" aria-describedby="chart-desc" />
<p id="chart-desc">详细描述...</p>
```
