HTML5 引入了一系列语义化标签，让网页结构更加清晰。这些标签不只是给页面换了个名字，它们承载着明确的语义，有助于搜索引擎理解内容、屏幕阅读器导航、开发者维护代码。

## 为什么需要语义化标签

在 HTML5 之前，页面结构通常是这样的：

```html
<div id="header">
  <div id="nav">...</div>
</div>
<div id="main">
  <div class="article">...</div>
  <div class="sidebar">...</div>
</div>
<div id="footer">...</div>
```

问题在于：浏览器和搜索引擎只看到一堆 `<div>`，无法理解哪部分是导航、哪部分是正文、哪部分是侧边栏。

HTML5 的语义化标签解决了这个问题：

```html
<header>
  <nav>...</nav>
</header>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>
```

## header 页头

`<header>` 表示页面或区块的头部，通常包含标题、Logo、导航等：

```html
<!-- 页面头部 -->
<header>
  <img src="logo.png" alt="网站 Logo" />
  <h1>网站名称</h1>
  <nav>
    <a href="/">首页</a>
    <a href="/about">关于</a>
  </nav>
</header>

<!-- 文章头部 -->
<article>
  <header>
    <h2>文章标题</h2>
    <p>作者：张三 | 发布于 2024-01-15</p>
  </header>
  <p>文章内容...</p>
</article>
```

### header 的特点

- 一个页面可以有多个 `<header>`
- 可以嵌套在 `<article>`、`<section>` 中作为区块的头部
- 不能嵌套在 `<footer>`、`<address>` 或另一个 `<header>` 中

### header vs h1-h6

`<header>` 是容器，`<h1>`-`<h6>` 是标题。它们的关系：

```html
<header>
  <h1>网站标题</h1>
  <!-- h1 在 header 里面 -->
  <p>网站副标题</p>
</header>
```

## nav 导航

`<nav>` 表示导航区域，包含页面主要的导航链接：

```html
<nav>
  <ul>
    <li><a href="/">首页</a></li>
    <li><a href="/products">产品</a></li>
    <li><a href="/about">关于我们</a></li>
    <li><a href="/contact">联系方式</a></li>
  </ul>
</nav>
```

### 什么时候用 nav

不是所有链接组都需要 `<nav>`，只有主要导航才需要：

```html
<!-- 应该用 nav：主导航 -->
<nav aria-label="主导航">
  <ul>
    <li><a href="/">首页</a></li>
    <li><a href="/about">关于</a></li>
  </ul>
</nav>

<!-- 应该用 nav：面包屑 -->
<nav aria-label="面包屑">
  <ol>
    <li><a href="/">首页</a></li>
    <li><a href="/products">产品</a></li>
    <li>当前页面</li>
  </ol>
</nav>

<!-- 不需要 nav：页脚链接 -->
<footer>
  <ul>
    <li><a href="/privacy">隐私政策</a></li>
    <li><a href="/terms">服务条款</a></li>
  </ul>
</footer>
```

### 多个 nav

页面可以有多个 `<nav>`，用 `aria-label` 区分：

```html
<header>
  <nav aria-label="主导航">...</nav>
</header>

<aside>
  <nav aria-label="侧边导航">...</nav>
</aside>

<footer>
  <nav aria-label="页脚导航">...</nav>
</footer>
```

## main 主内容

`<main>` 表示页面的主要内容，每个页面只能有一个：

```html
<body>
  <header>...</header>
  <nav>...</nav>

  <main>
    <!-- 页面的核心内容 -->
    <h1>文章标题</h1>
    <p>文章内容...</p>
  </main>

  <footer>...</footer>
</body>
```

### main 的规则

1. **唯一性**：每个页面只能有一个可见的 `<main>`
2. **不能嵌套**：不能放在 `<article>`、`<aside>`、`<footer>`、`<header>`、`<nav>` 中
3. **排除重复内容**：导航、版权信息等不应该在 `<main>` 中

```html
<!-- 错误：main 在 article 里面 -->
<article>
  <main>...</main>
</article>

<!-- 正确 -->
<main>
  <article>...</article>
</main>
```

### 跳过导航链接

`<main>` 配合跳转链接，帮助键盘用户：

```html
<a href="#main-content" class="skip-link">跳转到主内容</a>

<header>长长的导航...</header>

<main id="main-content">主内容</main>
```

```css
.skip-link {
  position: absolute;
  left: -9999px;
}

.skip-link:focus {
  left: 10px;
  top: 10px;
}
```

## article 文章

`<article>` 表示独立的、可复用的内容块：

```html
<article>
  <header>
    <h2>如何学习前端开发</h2>
    <time datetime="2024-01-15">2024年1月15日</time>
  </header>

  <p>前端开发是一个不断发展的领域...</p>

  <footer>
    <p>标签：前端、学习</p>
  </footer>
</article>
```

### 什么内容适合用 article

判断标准：**这段内容脱离当前页面后，是否仍然有意义？**

适合用 `<article>` 的：

- 博客文章
- 新闻报道
- 论坛帖子
- 用户评论
- 产品卡片

```html
<!-- 博客文章 -->
<article>
  <h2>文章标题</h2>
  <p>文章内容...</p>
</article>

<!-- 评论也是 article -->
<article>
  <header>
    <img src="avatar.jpg" alt="用户头像" />
    <span>用户名</span>
    <time>1小时前</time>
  </header>
  <p>评论内容...</p>
</article>

<!-- 产品卡片 -->
<article class="product-card">
  <img src="product.jpg" alt="产品图片" />
  <h3>产品名称</h3>
  <p>¥99.00</p>
</article>
```

### article 的嵌套

文章中的评论可以嵌套：

```html
<article>
  <h2>文章标题</h2>
  <p>文章内容...</p>

  <section>
    <h3>评论</h3>

    <article>
      <p>第一条评论</p>
    </article>

    <article>
      <p>第二条评论</p>
    </article>
  </section>
</article>
```

## section 章节

`<section>` 表示文档中的一个章节或区块，通常有标题：

```html
<article>
  <h1>HTML5 教程</h1>

  <section>
    <h2>简介</h2>
    <p>HTML5 是...</p>
  </section>

  <section>
    <h2>新特性</h2>
    <p>HTML5 新增了...</p>
  </section>

  <section>
    <h2>浏览器支持</h2>
    <p>现代浏览器都支持...</p>
  </section>
</article>
```

### section vs article vs div

| 元素        | 用途                   |
| ----------- | ---------------------- |
| `<article>` | 独立的、可复用的内容   |
| `<section>` | 文档的章节，通常有标题 |
| `<div>`     | 纯样式分组，没有语义   |

决策流程：

1. 内容可以独立存在吗？→ `<article>`
2. 内容是文档的一个主题章节吗？→ `<section>`
3. 只是为了样式分组？→ `<div>`

```html
<!-- section：教程的章节 -->
<section>
  <h2>第一章：基础</h2>
  ...
</section>

<!-- article：独立的博客文章 -->
<article>
  <h2>如何使用 section</h2>
  ...
</article>

<!-- div：纯样式包装 -->
<div class="container">
  <div class="row">...</div>
</div>
```

### section 必须有标题

每个 `<section>` 通常应该有标题（h1-h6）：

```html
<!-- 推荐 -->
<section>
  <h2>章节标题</h2>
  <p>内容...</p>
</section>

<!-- 不推荐：没有标题 -->
<section>
  <p>内容...</p>
</section>
```

## aside 侧边内容

`<aside>` 表示与主内容相关但可以独立存在的内容：

```html
<main>
  <article>
    <h1>文章标题</h1>
    <p>文章内容...</p>
  </article>

  <aside>
    <h2>相关文章</h2>
    <ul>
      <li><a href="#">相关文章1</a></li>
      <li><a href="#">相关文章2</a></li>
    </ul>
  </aside>
</main>
```

### aside 的用途

```html
<!-- 侧边栏 -->
<aside class="sidebar">
  <section>
    <h3>热门文章</h3>
    <ul>
      ...
    </ul>
  </section>
  <section>
    <h3>标签云</h3>
    <ul>
      ...
    </ul>
  </section>
</aside>

<!-- 文章内的旁注 -->
<article>
  <p>JavaScript 是一门...</p>
  <aside>
    <p>小知识：JavaScript 最初只用了10天开发完成。</p>
  </aside>
  <p>继续正文...</p>
</article>

<!-- 广告 -->
<aside>
  <a href="...">
    <img src="ad.jpg" alt="广告" />
  </a>
</aside>
```

## footer 页脚

`<footer>` 表示页面或区块的底部：

```html
<!-- 页面页脚 -->
<footer>
  <p>© 2024 网站名称. 保留所有权利。</p>
  <nav>
    <a href="/privacy">隐私政策</a>
    <a href="/terms">服务条款</a>
  </nav>
</footer>

<!-- 文章页脚 -->
<article>
  <h2>文章标题</h2>
  <p>文章内容...</p>
  <footer>
    <p>作者：张三</p>
    <p>标签：HTML, 前端</p>
  </footer>
</article>
```

### footer 的内容

典型的页脚内容：

- 版权信息
- 作者信息
- 相关链接
- 联系方式
- 社交媒体链接

```html
<footer>
  <div class="footer-links">
    <section>
      <h4>关于我们</h4>
      <ul>
        <li><a href="/about">公司介绍</a></li>
        <li><a href="/team">团队</a></li>
      </ul>
    </section>
    <section>
      <h4>支持</h4>
      <ul>
        <li><a href="/help">帮助中心</a></li>
        <li><a href="/contact">联系我们</a></li>
      </ul>
    </section>
  </div>

  <div class="footer-bottom">
    <p>© 2024 公司名称</p>
    <div class="social-links">
      <a href="...">微博</a>
      <a href="...">微信</a>
    </div>
  </div>
</footer>
```

## 其他语义标签

### figure 和 figcaption

```html
<figure>
  <img src="chart.png" alt="数据图表" />
  <figcaption>图1：2024年用户增长趋势</figcaption>
</figure>

<figure>
  <pre><code>console.log('Hello')</code></pre>
  <figcaption>示例代码</figcaption>
</figure>
```

### details 和 summary

```html
<details>
  <summary>点击查看详情</summary>
  <p>这里是详细内容，点击标题后展开。</p>
</details>

<details open>
  <summary>默认展开</summary>
  <p>这段内容默认是展开的。</p>
</details>
```

### time

```html
<p>发布于 <time datetime="2024-01-15">2024年1月15日</time></p>
<p>会议在 <time datetime="14:00">下午两点</time> 开始</p>
<p>持续 <time datetime="PT2H30M">2小时30分钟</time></p>
```

### address

```html
<address>
  联系作者：<a href="mailto:author@example.com">author@example.com</a><br />
  地址：北京市朝阳区
</address>
```

### mark

```html
<p>搜索结果中<mark>关键词</mark>会被高亮显示。</p>
```

## 完整页面结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>页面标题</title>
  </head>
  <body>
    <header>
      <a href="/" class="logo">Logo</a>
      <nav aria-label="主导航">
        <ul>
          <li><a href="/">首页</a></li>
          <li><a href="/blog">博客</a></li>
          <li><a href="/about">关于</a></li>
        </ul>
      </nav>
    </header>

    <main>
      <article>
        <header>
          <h1>文章标题</h1>
          <p>
            <time datetime="2024-01-15">2024年1月15日</time>
            · 作者：张三
          </p>
        </header>

        <section>
          <h2>引言</h2>
          <p>文章开头...</p>
        </section>

        <section>
          <h2>正文</h2>
          <p>主要内容...</p>

          <figure>
            <img src="example.jpg" alt="示例图片" />
            <figcaption>图1：示例</figcaption>
          </figure>
        </section>

        <footer>
          <p>标签：HTML, 语义化</p>
        </footer>
      </article>

      <aside>
        <section>
          <h2>相关文章</h2>
          <ul>
            <li><a href="#">相关文章1</a></li>
            <li><a href="#">相关文章2</a></li>
          </ul>
        </section>
      </aside>
    </main>

    <footer>
      <p>© 2024 网站名称</p>
      <nav aria-label="页脚导航">
        <a href="/privacy">隐私政策</a>
        <a href="/terms">服务条款</a>
      </nav>
    </footer>
  </body>
</html>
```

## 小结

HTML5 语义化标签让页面结构更清晰：

- **header**：页面或区块的头部
- **nav**：主要导航
- **main**：页面主内容（唯一���
- **article**：独立的、可复用的内容
- **section**：文档的章节
- **aside**：侧边内容
- **footer**：页面或区块的底部

使用语义化标签的好处：

- 代码可读性更好
- SEO 更友好
- 无障碍访问更好
- 便于维护
