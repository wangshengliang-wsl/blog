SEO（Search Engine Optimization，搜索引擎优化）是让网站在搜索结果中获得更好排名的技术。HTML 是 SEO 的基础，正确的 HTML 结构能帮助搜索引擎理解和索引你的内容。

## 搜索引擎如何理解页面

搜索引擎通过爬虫程序访问网页，分析 HTML 结构和内容。它们不看 CSS 样式，主要关注：

- 页面标题（title）
- 标题结构（h1-h6）
- 正文内容
- 链接关系
- Meta 信息
- 结构化数据

好的 HTML 结构让搜索引擎能准确理解页面主题和内容层级。

## title 标签

`<title>` 是 SEO 最重要的标签之一，显示在搜索结果中：

```html
<head>
  <title>文章标题 - 网站名称</title>
</head>
```

### 标题最佳实践

```html
<!-- 好的标题 -->
<title>JavaScript 数组方法详解 - 前端教程</title>
<title>2024年最佳笔记本电脑推荐 | TechReview</title>
<title>北京三日游攻略：景点、美食、住宿全指南</title>

<!-- 不好的标题 -->
<title>首页</title>
<title>文章</title>
<title>关于我们 - 关于我们 - 关于我们</title>
```

标题要求：

- 长度控制在 50-60 个字符
- 包含目标关键词
- 每个页面标题唯一
- 关键词靠前
- 网站名放在最后

### 动态标题

```javascript
// 根据内容动态设置标题
document.title = `${articleTitle} - ${siteName}`

// 不同页面的标题模式
// 首页：品牌名 - 简短描述
// 列表：分类名 - 品牌名
// 详情：内容标题 - 品牌名
```

## Meta 描述

`<meta name="description">` 显示在搜索结果的描述区域：

```html
<head>
  <meta
    name="description"
    content="本文详细介绍 JavaScript 数组的常用方法，包括 map、filter、reduce 等，配有完整示例代码。"
  />
</head>
```

### 描述最佳实践

```html
<!-- 好的描述 -->
<meta
  name="description"
  content="学习 10 个最常用的 JavaScript 数组方法，包括详细解释和实际应用示例。适合前端初学者。"
/>

<!-- 不好的描述 -->
<meta name="description" content="JavaScript,数组,方法,教程" />
<meta name="description" content="欢迎来到我们的网站，这里有很多内容..." />
```

描述要求：

- 长度 150-160 个字符
- 准确概括页面内容
- 包含目标关键词
- 每个页面描述唯一
- 有吸引力，鼓励点击

## 标题层级

搜索引擎通过 h1-h6 理解内容结构：

```html
<!-- 正确的标题层级 -->
<h1>JavaScript 数组方法详解</h1>

<h2>遍历方法</h2>
<h3>forEach</h3>
<h3>map</h3>
<h3>filter</h3>

<h2>查找方法</h2>
<h3>find</h3>
<h3>includes</h3>

<h2>转换方法</h2>
<h3>reduce</h3>
<h3>join</h3>
```

### 标题 SEO 要点

```html
<!-- 每个页面只有一个 h1 -->
<h1>页面主标题��包含核心关键词</h1>

<!-- h1 不要太长 -->
<h1>JavaScript 数组方法完整指南</h1>
<!-- 好 -->
<h1>这是一篇关于 JavaScript 中数组各种方法的详细介绍文章...</h1>
<!-- 太长 -->

<!-- 不要跳过层级 -->
<h1>标题</h1>
<h3>子标题</h3>
<!-- 错误：跳过了 h2 -->

<h1>标题</h1>
<h2>子标题</h2>
<!-- 正确 -->
```

## 语义化标签

语义化标签帮助搜索引擎理解页面结构：

```html
<!DOCTYPE html>
<html lang="zh-Hans">
  <head>
    <title>文章标题 - 网站名</title>
    <meta name="description" content="文章描述" />
  </head>
  <body>
    <header>
      <nav>主导航</nav>
    </header>

    <main>
      <article>
        <header>
          <h1>文章标题</h1>
          <time datetime="2024-01-15">2024年1月15日</time>
        </header>

        <p>文章内容...</p>

        <footer>
          <p>作者：张三</p>
        </footer>
      </article>
    </main>

    <aside>
      <h2>相关文章</h2>
      <ul>
        ...
      </ul>
    </aside>

    <footer>
      <p>版权信息</p>
    </footer>
  </body>
</html>
```

### 重要语义标签

| 标签        | SEO 作用                     |
| ----------- | ---------------------------- |
| `<article>` | 标识独立内容，可能被单独索引 |
| `<nav>`     | 标识导航区域                 |
| `<main>`    | 标识主要内容                 |
| `<aside>`   | 标识辅助内容                 |
| `<time>`    | 标识日期时间                 |
| `<address>` | 标识联系信息                 |

## 链接优化

### 内部链接

```html
<!-- 使用描述性的锚文本 -->
<a href="/javascript-array">JavaScript 数组方法详解</a>
<!-- ��� -->
<a href="/javascript-array">点击这里</a>
<!-- 不好 -->

<!-- 相关内容链接 -->
<article>
  <p>关于数组的更多内容，请参考 <a href="/array-basics">数组基础教程</a>。</p>
</article>

<!-- 面包屑导航 -->
<nav aria-label="面包屑">
  <ol itemscope itemtype="https://schema.org/BreadcrumbList">
    <li
      itemprop="itemListElement"
      itemscope
      itemtype="https://schema.org/ListItem"
    >
      <a itemprop="item" href="/"><span itemprop="name">首页</span></a>
      <meta itemprop="position" content="1" />
    </li>
    <li
      itemprop="itemListElement"
      itemscope
      itemtype="https://schema.org/ListItem"
    >
      <a itemprop="item" href="/tutorials"><span itemprop="name">教程</span></a>
      <meta itemprop="position" content="2" />
    </li>
    <li
      itemprop="itemListElement"
      itemscope
      itemtype="https://schema.org/ListItem"
    >
      <span itemprop="name">JavaScript 数组</span>
      <meta itemprop="position" content="3" />
    </li>
  </ol>
</nav>
```

### 外部链接

```html
<!-- 外部链接添加 rel 属性 -->
<a href="https://external.com" rel="noopener noreferrer" target="_blank"
  >外部网站</a
>

<!-- 付费链接或广告 -->
<a href="https://sponsor.com" rel="sponsored">赞助商</a>

<!-- 用户生成的链接 -->
<a href="https://user-link.com" rel="ugc">用户提交的链接</a>

<!-- 不想传递权重的链接 -->
<a href="https://example.com" rel="nofollow">不信任的链接</a>
```

### 规范链接

指定页面的权威版本：

```html
<head>
  <!-- 规范 URL -->
  <link rel="canonical" href="https://example.com/article" />

  <!-- 多语言版本 -->
  <link rel="alternate" hreflang="en" href="https://example.com/en/article" />
  <link rel="alternate" hreflang="zh" href="https://example.com/zh/article" />

  <!-- 分页 -->
  <link rel="prev" href="https://example.com/page/1" />
  <link rel="next" href="https://example.com/page/3" />
</head>
```

## 图片 SEO

### alt 属性

```html
<!-- 描述性的 alt -->
<img src="chart.png" alt="2024年各季度销售数据柱状图，Q4最高达150万" />

<!-- 包含关键词，但自然 -->
<img src="macbook.jpg" alt="MacBook Pro 16英寸笔记本电脑正面展示" />

<!-- 装饰性图片用空 alt -->
<img src="decoration.png" alt="" />
```

### 图片文件名

```html
<!-- 好的文件名 -->
<img src="javascript-array-methods.png" alt="JavaScript 数组方法" />
<img src="macbook-pro-16-review.jpg" alt="MacBook Pro 16评测" />

<!-- 不好的文件名 -->
<img src="IMG_001.jpg" alt="..." />
<img src="photo.png" alt="..." />
```

### 图片懒加载

```html
<!-- 原生懒加载 -->
<img src="image.jpg" alt="描述" loading="lazy" />

<!-- 重要图片不要懒加载 -->
<img src="hero.jpg" alt="主图" loading="eager" />
```

## 结构化数据

结构化数据帮助搜索引擎理解内容类型，可能在搜索结果中显示富摘要。

### JSON-LD 格式

```html
<head>
  <script type="application/ld+json">
    {
      "@context": "https://schema.org",
      "@type": "Article",
      "headline": "JavaScript 数组方法详解",
      "description": "本文详细介绍 JavaScript 数组的常用方法",
      "author": {
        "@type": "Person",
        "name": "张三"
      },
      "datePublished": "2024-01-15",
      "dateModified": "2024-01-20",
      "publisher": {
        "@type": "Organization",
        "name": "前端教程",
        "logo": {
          "@type": "ImageObject",
          "url": "https://example.com/logo.png"
        }
      },
      "image": "https://example.com/article-image.png"
    }
  </script>
</head>
```

### 常见结构化数据类型

#### 文章

```html
<script type="application/ld+json">
  {
    "@context": "https://schema.org",
```

```html
    "@type": "Product",
    "name": "产品名称",
    "image": "https://example.com/product.jpg",
```

## 技术 SEO

### robots 标签

```html
<head>
  <!-- 允许索引 -->
  <meta name="robots" content="index, follow" />

  <!-- 禁止索引 -->
  <meta name="robots" content="noindex, nofollow" />

  <!-- 禁止显示缓存快照 -->
  <meta name="robots" content="noarchive" />

  <!-- 禁止显示摘要 -->
  <meta name="robots" content="nosnippet" />
</head>
```

### 移动端优化

```html
<head>
  <!-- 视口设置 -->
  <meta name="viewport" content="width=device-width, initial-scale=1" />

  <!-- 告诉搜索引擎这是移动友好的 -->
  <meta name="mobile-web-app-capable" content="yes" />
</head>
```

### 页面速度

```html
<!-- 预加载关键资源 -->
<link rel="preload" href="critical.css" as="style" />
<link rel="preload" href="main.js" as="script" />

<!-- DNS 预解析 -->
<link rel="dns-prefetch" href="//cdn.example.com" />

<!-- 预连接 -->
<link rel="preconnect" href="https://api.example.com" />

<!-- 图片懒加载 -->
<img src="image.jpg" loading="lazy" alt="描述" />

<!-- 异步加载非关键脚本 -->
<script src="analytics.js" async></script>
<script src="widget.js" defer></script>
```

```markdown

- [ ] 每个页面有唯一的 title
- [ ] 每个页面有 meta description

## 内容检查

- [ ] 标题层级正确（不跳级）
- [ ] 所有图片有 alt 属性
- [ ] 链接使用描述性文本
- [ ] 内容原创且有价值

## 技术检查

- [ ] 页面加载速度快
- [ ] 移动端友好
- [ ] 无死链
- [ ] 有 canonical 标签
- [ ] 结构化数据正确

## 社交分享

- [ ] 有 Open Graph 标签
- [ ] 有 Twitter Card 标签
- [ ] 分享图片尺寸正确
```

## 常见 SEO 错误

### 1. 重复内容

```html
<!-- 错误：多个页面相同内容，没有 canonical -->
<!-- page1.html 和 page2.html 内容相同 -->

<!-- 正确：指定规范版本 -->
<link rel="canonical" href="https://example.com/page1.html" />
```

### 2. 隐藏内容

```html
<!-- 不要隐藏关键内容来欺骗搜索引擎 -->
<p style="display:none;">关键词堆砌...</p>
<p style="font-size:0;">更多关键词...</p>
```

### 3. 关键词堆砌

```html
<!-- 不好：关键词堆砌 -->
<title>JavaScript JavaScript教程 JavaScript学习 JavaScript入门</title>

<!-- 好：自然使用关键词 -->
<title>JavaScript 入门教程 - 从零开始学习</title>
```

### 4. 忽略移动端

```html
<!-- 确保移动端友好 -->
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

## 小结

HTML 对 SEO 的影响主要体现在：

**基础标签**：

- `<title>`：最重要的 SEO 标签
- `<meta description>`：影响点击率
- `<h1>`-`<h6>`：内容结构

**语义化**：

- 使用正确的语义标签
- 帮助搜索引擎理解内容
- 提供结构化数据

**链接优化**：

- 描述性的锚文本
- 正确使用 rel 属性
- 设置 canonical 标签

**技术优化**：

- 页面加载速度
- 移动端友好
- 图片优化

记住：SEO 的本质是提供有价值的内容和良好的用户体验，HTML 只是载体。
