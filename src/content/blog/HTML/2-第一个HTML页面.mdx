上一篇介绍了 HTML 的基本概念。这篇文章来详细拆解一个完整的 HTML 页面结构，理解每一部分的作用。

## HTML 文档的基本结构

打开任何一个网页的源代码，都会看到类似这样的骨架：

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>页面标题</title>
  </head>
  <body>
    <!-- 页面内容 -->
  </body>
</html>
```

这个结构可以分为四个核心部分：

| 部分              | 作用                 |
| ----------------- | -------------------- |
| `<!DOCTYPE html>` | 文档类型声明         |
| `<html>`          | 根元素，包含整个页面 |
| `<head>`          | 头部，存放元数据     |
| `<body>`          | 主体，存放可见内容   |

下面逐一讲解。

## DOCTYPE 文档类型声明

```html
<!DOCTYPE html>
```

这行代码必须放在 HTML 文件的最开头，它告诉浏览器：这是一个 HTML5 文档，请用标准模式来渲染。

### 为什么需要 DOCTYPE

早期浏览器对 HTML 的解析方式不统一。为了兼容旧网页，浏览器有两种渲染模式：

- **标准模式（Standards Mode）**：按照 W3C 标准渲染
- **怪异模式（Quirks Mode）**：模拟旧浏览器的行为

没有 DOCTYPE 或者 DOCTYPE 格式不对，浏览器就会进入怪异模式，导致一些 CSS 属性表现异常。比如在怪异模式下，盒模型的计算方式不同，元素的实际宽度会和预期不一致。

### DOCTYPE 的历史演变

HTML4 和 XHTML 的 DOCTYPE 非常冗长：

```html
<!-- HTML 4.01 Strict -->
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<!-- XHTML 1.0 Strict -->
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
```

HTML5 简化成了 `<!DOCTYPE html>`，不再需要引用 DTD（Document Type Definition）。这种简洁的写法兼容所有现代浏览器，也能触发标准模式。

### DOCTYPE 的注意事项

1. **必须放在第一行**：DOCTYPE 前面不能有任何内容，包括空格、注释、BOM 标记
2. **大小写不敏感**：`<!doctype html>`、`<!DOCTYPE HTML>` 都有效，但习惯上写成 `<!DOCTYPE html>`
3. **不是 HTML 标签**：DOCTYPE 是一种声明，不需要闭合

## html 根元素

```html
<html lang="zh-CN">
  <!-- 所有其他元素都在这里面 -->
</html>
```

`<html>` 是 HTML 文档的根元素，所有其他元素都是它的后代。一个 HTML 文件只能有一个 `<html>` 元素。

### lang 属性

`lang` 属性指定页面的主要语言。这个属性虽然不影响页面显示，但非常重要：

| 用途       | 说明                               |
| ---------- | ---------------------------------- |
| 搜索引擎   | 根据语言为用户返回合适的结果       |
| 屏幕阅读器 | 选择正确的发音和语调               |
| 浏览器翻译 | 判断是否需要提示翻译               |
| CSS 选择器 | `:lang()` 伪类可以根据语言应用样式 |

常用的语言代码：

```html
<html lang="zh-CN">
  <!-- 简体中文 -->
  <html lang="zh-TW">
    <!-- 繁体中文 -->
    <html lang="en">
      <!-- 英语 -->
      <html lang="ja">
        <!-- 日语 -->
      </html>
    </html>
  </html>
</html>
```

如果页面包含多种语言的内容，可以在特定元素上单独指定：

```html
<html lang="zh-CN">
  <body>
    <p>这是中文内容。</p>
    <p lang="en">This is English content.</p>
  </body>
</html>
```

## head 头部元素

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>页面标题</title>
  <meta name="description" content="页面描述" />
  <link rel="stylesheet" href="style.css" />
  <script src="app.js"></script>
</head>
```

`<head>` 元素包含页面的**元数据**（metadata），这些信息不会直接显示在页面上，但对浏览器、搜索引擎���其他机器来说非常重要。

### title 标题

```html
<title>页面标题 - 网站名称</title>
```

`<title>` 定义页面的标题，它会显示在：

- 浏览器标签页上
- 浏览器历史记录和书签中
- 搜索引擎结果页面（SERP）

每个页面都必须有且只有一个 `<title>` 元素。标题应该简洁有意义，一般建议控制在 60 个字符以内，超出的部分在搜索结果中会被截断。

### meta 元数据

`<meta>` 标签用于定义各种元数据，格式通常是 `name` + `content` 或 `charset`。

#### 字符编码

```html
<meta charset="UTF-8" />
```

指定页面使用 UTF-8 编码。UTF-8 可以表示世界上几乎所有的字符，是网页的首选编码。这个标签应该尽量靠近 `<head>` 开头，让浏览器尽早知道编码方式。

#### 视口设置

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

视口（viewport）设置对移动端适配至关重要：

- `width=device-width`：视口宽度等于设备宽度
- `initial-scale=1.0`：初始缩放比例为 1

没有这个设置，移动设备会假设页面是为桌面设计的，导致页面被缩小显示。

#### 页面描述

```html
<meta name="description" content="这里是页面的简短描述，会显示在搜索结果中" />
```

`description` 会被搜索引擎���作搜索结果的摘要。一般建议控制在 150-160 个字符，描述应该准确概括页面内容，吸引用户点击。

#### 关键词（已过时）

```html
<meta name="keywords" content="HTML, 教程, 前端" />
```

曾经用于 SEO，但由于被滥用，现代搜索引擎已经忽略这个标签。不写也没关系。

#### 作者

```html
<meta name="author" content="作者名" />
```

标明页面作者。

#### 机器人指令

```html
<meta name="robots" content="index, follow" />
<meta name="robots" content="noindex, nofollow" />
```

告诉搜索引擎爬虫如何处理页面：

- `index`：允许索引
- `noindex`：禁止索引
- `follow`：允许跟踪页面上的链接
- `nofollow`：禁止跟踪链接

#### HTTP 等效头

```html
<!-- 自动刷新页面 -->
<meta http-equiv="refresh" content="5;url=https://example.com" />

<!-- 内容安全策略 -->
<meta http-equiv="Content-Security-Policy" content="default-src 'self'" />
```

`http-equiv` 可以模拟 HTTP 响应头的效果。

### link 链接外部资源

```html
<!-- 外部样式表 -->
<link rel="stylesheet" href="style.css" />

<!-- 网站图标 -->
<link rel="icon" href="favicon.ico" />
<link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png" />

<!-- 预加载资源 -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin />

<!-- 规范链接 -->
<link rel="canonical" href="https://example.com/page" />
```

`<link>` 标签用于引入外部资源，`rel` 属性指定资源与页面的关系。

常用的 `rel` 值：

| rel 值       | 用途                         |
| ------------ | ---------------------------- |
| `stylesheet` | CSS 样式表                   |
| `icon`       | 网站图标（favicon）          |
| `preload`    | 预加载重要资源               |
| `prefetch`   | 预获取下个页面可能用到的资源 |
| `canonical`  | 指定规范 URL（避免重复内容） |
| `alternate`  | 替代版本（如 RSS、不同语言） |

### script 脚本

```html
<!-- 外部脚本 -->
<script src="app.js"></script>

<!-- 内联脚本 -->
<script>
  console.log('Hello, World!')
</script>
```

`<script>` 可以放在 `<head>` 或 `<body>` 中，位置会影响加载和执行时机。

#### 脚本的加载行为

默认情况下，浏览器遇到 `<script>` 会：

1. 暂停 HTML 解析
2. 下载脚本
3. 执行脚本
4. 继续解析 HTML

这会阻塞页面渲染。为了优化性能，可以使用 `async` 或 `defer` 属性：

```html
<!-- async：下载完立即执行，不保证顺序 -->
<script src="analytics.js" async></script>

<!-- defer：下载完后等 HTML 解析完再执行，保证顺序 -->
<script src="app.js" defer></script>
```

| 属性  | 下载时机   | 执行时机     | 执行顺序 |
| ----- | ---------- | ------------ | -------- |
| 无    | 立即，阻塞 | 立即         | 按顺序   |
| async | 并行       | 下载完立即   | 不保证   |
| defer | 并行       | DOM 解析完后 | 按顺序   |

#### type 属性

```html
<!-- ES 模块 -->
<script type="module" src="app.mjs"></script>

<!-- JSON 数据 -->
<script type="application/json" id="config">
  { "theme": "dark" }
</script>
```

`type="module"` 用于加载 ES 模块，自动启用严格模式，支持 `import`/`export`。

### style 内部样式

```html
<style>
  body {
    font-family: sans-serif;
    line-height: 1.6;
  }
</style>
```

`<style>` 用于定义内部样式。虽然方便，但通常推荐使用外部样式表，便于缓存和维护。

### base 基准 URL

```html
<base href="https://example.com/" target="_blank" />
```

`<base>` 为页面上所有相对 URL 指定基准地址，`target` 为所有链接指定默认打开方式。一个页面只能有一个 `<base>` 元素，且必须在任何带有 URL 的元素之前。

这个标签很少使用，因为它会影响页面上所有的相对链接，容易造成意外。

## body 主体元素

```html
<body>
  <header>页头</header>
  <main>
    <h1>主标题</h1>
    <p>正文内容</p>
  </main>
  <footer>页脚</footer>
</body>
</html>
```

`<body>` 包含页面的所有可见内容：文本、图片、链接、表单……用户看到的一切都在 `<body>` 里。

### body 的属性

HTML5 之前，`<body>` 有一些控制页面外观的属性，如 `bgcolor`、`text`、`link`。现在这些都应该用 CSS 来实现：

```html
<!-- 旧的写法（不推荐） -->
<body bgcolor="#ffffff" text="#000000">
  <!-- 现代写法 -->
  <body>
    <!-- CSS -->
    <style>
      body {
        background-color: #ffffff;
        color: #000000;
      }
    </style>
  </body>
</body>
```

## 注释

```html
<!-- 这是一条注释 -->

<!--
  这是一条
  多行注释
-->
```

HTML 注释以 `<!--` 开始，以 `-->` 结束。注释不会显示在页面上，但会出现在源代码中。

注释的用途：

- 解释复杂的代码结构
- 临时禁用某段代码
- 标记代码区域（如 `<!-- 导航开始 -->`）

注意：不要在注释里放敏感信息，用户可以查看源代码。

## 一个完整的模板

把上面讲的内容整合起来，这是一个现代 HTML 页面的完整模板：

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <!-- 字符编码 -->
    <meta charset="UTF-8" />

    <!-- 视口设置 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- 页面标题 -->
    <title>页面标题 - 网站名称</title>

    <!-- SEO 元数据 -->
    <meta name="description" content="页面的简短描述" />
    <meta name="author" content="作者名" />

    <!-- Open Graph（社交分享） -->
    <meta property="og:title" content="分享标题" />
    <meta property="og:description" content="分享描述" />
    <meta property="og:image" content="https://example.com/image.jpg" />

    <!-- 网站图标 -->
    <link rel="icon" href="/favicon.ico" />

    <!-- 样式表 -->
    <link rel="stylesheet" href="/css/style.css" />
  </head>
  <body>
    <!-- 页头 -->
    <header>
      <nav>导航</nav>
    </header>

    <!-- 主要内容 -->
    <main>
      <h1>页面主标题</h1>
      <article>
        <p>正文内容</p>
      </article>
    </main>

    <!-- 页脚 -->
    <footer>
      <p>版权信息</p>
    </footer>

    <!-- 脚本 -->
    <script src="/js/app.js" defer></script>
  </body>
</html>
```

## 常见问题

### 标签一定要闭合吗

HTML5 对标签闭合比较宽容：

```html
<!-- 这些写法都有效 -->
<p>
  段落内容
  <br />
  <img src="photo.jpg" alt="照片" />

  <!-- XHTML 风格（也有效） -->
</p>

<p>段落内容</p>
<br />
<img src="photo.jpg" alt="照片" />
```

但为了代码清晰和避免潜在问题，建议：

- 非空元素（如 `<p>`、`<div>`）要写结束标签
- 空元素（如 `<br>`、`<img>`）可以不加 `/`

### 标签名大小写

HTML5 不区分大小写，`<DIV>`、`<Div>`、`<div>` 都有效。但习惯上**全部使用小写**，这也是代码规范的要求。

### 属性值的引号

属性值可以用双引号、单引号，甚至不用引号（如果值没有空格等特殊字符）：

```html
<!-- 都有效 -->
<div class="container">
  <div class="container">
    <div class="container"></div>
  </div>
</div>
```

推荐使用**双引号**，这是最常见的写法。

## 小结

这篇文章详细讲解了 HTML 页面的基本结构：

- `<!DOCTYPE html>` 声明文档类型，触发标准模式
- `<html>` 是根元素，`lang` 属性指定语言
- `<head>` 包含元数据：标题、字符编码、视口、样式、脚本
- `<body>` 包含所有可见内容
