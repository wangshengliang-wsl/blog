## 面试题

> 你平时开发 Vue 项目时，做了哪些性能优化？

## 一、代码层面

| 优化点         | 说明                                 |
| -------------- | ------------------------------------ |
| 组件拆分       | 细粒度组件，减少重渲染范围           |
| 减少响应式数据 | 不需要响应式的数据用 `Object.freeze` |
| v-if vs v-show | 频繁切换用 v-show，条件稳定用 v-if   |
| v-once         | 只渲染一次的静态内容                 |
| key 优化       | 列表使用唯一 key                     |
| 防抖节流       | 高频事件处理                         |
| KeepAlive      | 缓存不活跃的组件                     |
| 异步组件       | 按需加载组件                         |
| 路由懒加载     | 按需加载路由                         |

## 二、场景优化

| 场景       | 方案            |
| ---------- | --------------- |
| 长列表     | 虚拟滚动        |
| 图片多     | 图片懒加载      |
| 大数据表格 | 分页 / 虚拟表格 |

## 三、静态资源

| 优化点   | 方案                |
| -------- | ------------------- |
| 图片     | 压缩、WebP 格式     |
| 图标     | SVG 图标 / 字体图标 |
| JS/CSS   | 压缩、合并          |
| 第三方库 | CDN 加速            |

## 四、打包优化

| 优化点       | 说明                     |
| ------------ | ------------------------ |
| Tree Shaking | 移除未使用代码           |
| 代码分割     | 按需加载                 |
| 第三方库     | 按需引入（如 lodash-es） |
| 多线程打包   | thread-loader            |
| 构建工具     | Vite 替代 Webpack        |

## 五、用户体验

| 优化点   | 说明           |
| -------- | -------------- |
| 骨架屏   | 加载时显示占位 |
| 加载动画 | 提供视觉反馈   |
| 过渡动画 | 平滑的状态切换 |
| SSR/SSG  | 首屏渲染优化   |

## 性能问题排查思路

1. **发现问题**：使用 Lighthouse、Chrome DevTools 获取数据
2. **确定问题来源**：前端、网络、服务端、数据库？
3. **针对性优化**：根据具体瓶颈采取对应措施

## 参考答案

性能优化分几个维度：

**代码层面**：组件拆分、v-if/v-show 正确使用、key 优化、异步组件、路由懒加载等。

**场景优化**：长列表用虚拟滚动，图片用懒加载。

**静态资源**：图片压缩、使用 WebP、SVG 图标、CDN 加速。

**打包优化**：Tree Shaking、代码分割、按需引入第三方库。

**用户体验**：骨架屏、加载动画、过渡效果。

所有优化的最终目的都是提升用户体验。
