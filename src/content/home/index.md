## 关于我

哈喽👋，我是 **`wangshengliang`**，一个热爱折腾代码的前端工程师。

## 我的日常

写代码这件事，说实话就像在玩一个永远通不完关的游戏。早上可能还在为一个奇怪的 CSS 布局抓耳挠腮，下午就被 TypeScript 的类型体操搞得头晕眼花，晚上又开始研究最新的 React Server Components。每个 bug 都是一个小 boss，解决掉的瞬间那种成就感，比通关游戏还爽。

这些年来，我倒腾过不少东西：从 0 到 1 搭建过电商平台，优化过让人头秃的管理后台，也给团队搭建过不少基础设施。有时候为了一个丝滑的交互动画，能在 Chrome DevTools 里反复调试到深夜；有时候为了减少几 KB 的包体积，会翻遍 webpack 文档找最佳实践。同事们都说我有点"完美主义"，但我更愿意理解为——对用户体验的偏执。

## 技术栈这点事

主要玩 **React** 和 **Next.js** 这套，TypeScript 用得多了也就习惯了它的"严格"。最近在深入 **Server Components** 和边缘计算，感觉前端的边界正在被不断拓宽。后端的话，Node.js + PostgreSQL 是常规配置，偶尔也会折腾一下 Serverless 和 Cloudflare Workers。

工具链方面，Tailwind CSS 让我彻底告别了 CSS 命名焦虑，Drizzle ORM 用起来比想象中顺手，Biome 的速度确实比 ESLint 快不少。对了，还有那些让开发效率翻倍的小工具——Raycast、VS Code 插件、各种 CLI...总之，能自动化的绝不手动。

## 团队与分享

我挺享受和同事们分享技术方案的过程，有时候一个技术群里的随口讨论，就能演变成一次深度的架构重构。毕竟一个人写代码容易钻牛角尖，多几双眼睛、多几个脑子，问题往往就迎刃而解了。

开源社区也是我经常逛的地方，看到好的项目会 star，遇到 bug 偶尔也会提个 PR。这种"我为人人，人人为我"的氛围，大概就是开发者世界最迷人的地方吧。

## 为什么写博客

说起来建这个博客的初衷挺简单：一方面是想把开发过程中的思考、踩过的坑记录下来，方便自己以后查阅（毕竟记性没以前那么好了）；另一方面，如果这些经验能帮到正在解决类似问题的朋友，那就更有意义了。

当然，这里也欢迎大家随时来"拍砖"。技术这东西，没有绝对的对错，只有更适合的方案。我写的东西不一定都是最优解，如果你有更好的想法，咱们可以聊聊。

## 生活碎片

除了写代码，我也喜欢鼓捣一些"不务正业"的东西：研究唱歌技巧🎤，训练相对音感🎵，偶尔玩玩游戏放松一下🎮。对了，还有看技术书和纪录片🎬，感觉每次读都有新的体会。

生活嘛，总得有点代码之外的乐趣，这样写起代码来才更有灵感，你说是不是？
